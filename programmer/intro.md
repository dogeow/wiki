## 详细

- SOLID 原则：
  - 单一职责原则
  - 开闭原则
  - 里氏替换原则
  - 依赖反转原则
  - 接口隔离原则
- 坏味道（Code Smell）
  - 代码重复
  - 过大的函数 / 类 / 模块
  - 不恰当的命名
  - 不一致的风格
  - 不合理的复杂性
  - 过多的 if else
  - 过多的参数
  - 紧密耦合
  - 不恰当的注释或没有注释
- 其他原则
  - 合成复用原则
  - 迪米特法则

## 过程

1. 原型图、UI 图的查看
2. 需求反馈、确定
3. 工期评估，接口数量粗略统计
4. 逻辑的思维导图或 UML 图、数据库设计
5. 字段验证、权限控制
6. 代码编写、debug 时的日志编写
7. 接口可靠性冗余、高并发处理、频率控制
8. API 文档编写
9. 单元测试
10. CI / CD 集成
11. 设置服务监控

## 维护

1. 日志查看、服务监控的提醒
2. 代码重构

> 过程中包含：字段的英语单词选用
